<p>Bạn đã bao giờ thấy app crash hoặc show cryptic error messages? Hay user complain "app không hoạt động" mà không biết tại sao?</p>
<p>Network programming đầy rủi ro: connections drop, servers down, invalid responses. Nếu không handle errors properly, user experience terrible, và debugging nightmare.</p>
<p>Trong production app đầu tiên, tôi chỉ log errors to console. Users thấy blank screens, file complaints. Sau khi implement proper error handling, user satisfaction tăng đáng kể.</p>
<h2>Phân loại network errors</h2>
<p>Network errors chia thành categories khác nhau:</p>
<p><strong>Connection errors</strong>: Không thể reach server (network down, DNS issues)
<strong>HTTP errors</strong>: Server trả error codes (404, 500, etc.)
<strong>Timeout errors</strong>: Request quá thời gian
<strong>Data errors</strong>: Response malformed hoặc unexpected</p>
<pre><code class="language-javascript">async function robustFetch(url) {
  try {
    const response = await fetch(url, { timeout: 5000 });

    if (!response.ok) {
      // HTTP error
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }

    const data = await response.json();
    return data;

  } catch (error) {
    if (error.name === 'AbortError') {
      throw new Error('Request timeout - kiểm tra kết nối');
    } else if (error.message.includes('HTTP')) {
      throw error; // HTTP errors giữ nguyên
    } else {
      throw new Error('Network error - không thể kết nối server');
    }
  }
}
</code></pre>
<p>Handle từng loại error khác nhau, user biết exactly what happened.</p>

<h3>Error Handling Strategy</h3>
<p>Tạo error handler tập trung để phân loại và xử lý errors consistently:</p>
<pre><code class="language-javascript">class ErrorHandler {
  static handle(error) {
    if (error.name === 'TypeError') return { type: 'CONNECTION_ERROR', retryable: true };
    if (error.name === 'AbortError') return { type: 'TIMEOUT_ERROR', retryable: true };
    if (error.message.includes('HTTP')) return { type: 'HTTP_ERROR', retryable: error.status >= 500 };
    return { type: 'UNKNOWN_ERROR', retryable: true };
  }
}
</code></pre>
<h3>Retry Strategy</h3>
<p>Combine retry với error handling cho robustness:</p>
<pre><code class="language-javascript">async function fetchWithRetry(url, maxRetries = 3) {
  for (let i = 0; i &#x3C; maxRetries; i++) {
    try {
      const response = await fetch(url);
      if (response.ok) return await response.json();
      throw new Error(`HTTP ${response.status}`);
    } catch (error) {
      if (i === maxRetries - 1) throw error;
      // Wait before retry
      await new Promise(resolve => setTimeout(resolve, 1000 * (i + 1)));
    }
  }
}
</code></pre>

<p>Sau khi deploy error handling system toàn diện với logging, monitoring, và user-friendly messages, tôi thấy user complaints giảm drasticaly.</p>
<p>Thứ nhất, user experience cải thiện rõ rệt. Thay vì blank screens và cryptic errors, users thấy clear messages: "Mất kết nối mạng, vui lòng thử lại" thay vì "TypeError: Failed to fetch".</p>
<p>Thứ hai, debugging trở nên dễ dàng. Với centralized logging và error tracking, tôi có thể identify patterns: "sáng thứ 2 có nhiều timeout errors" → "server overloaded on Monday mornings".</p>
